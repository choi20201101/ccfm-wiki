# /feedfm — 퍼포먼스 캔버스 진단 피드백 생성기

응시자의 **퍼포먼스 캔버스 진단 답안 + 점수**를 입력받아, **변하정/신동협 양식과 100% 동일한** 다크테마 1페이지 피드백 문서(HTML + PDF)를 만든다.

핵심 가치 두 축:
1. **양식(비주얼)** — 검정 배경·흰 글씨·핑크 강조. `assets/template.html` 의 `<style>` 를 **절대 변형 없이** 사용.
2. **내용(채점 방법론)** — 아래 "캔버스 채점 루브릭"으로 답안을 일관되게 채점·코칭. 깡통 칭찬 금지, 감점 사유와 "한 단계 더" 처방을 항상 붙인다.

```
/feedfm                         # 직전 대화에 붙여넣은 진단 결과로 생성
/feedfm <결과.txt 경로>          # 파일에서 읽어 생성
/feedfm --out <폴더>            # 출력 위치 지정 (기본: 입력과 같은 폴더 / Desktop/feed)
```

---

## 0️⃣ 입력 (반드시 확보)

응시자 진단 결과 텍스트. 보통 다음을 포함:
- 응시자명, 제품명
- 종합 점수, A 랜딩 OX(n/13), B 본질 객관식(n/25), C 주관식 AI(n/23), 등급
- 문항별 답변 (특히 **주관식 14·15·21·22·23·24** 가 피드백의 핵심)

없으면 사용자에게 결과 텍스트를 달라고 1회 요청. 점수 칸 일부가 비면 가진 것만으로 진행하고 비는 칸은 "—" 표기.

> 이 스킬은 **주관식(C) 집중 분석**이 기본 모드다. A/B는 점수 카드 + 한 줄 요약으로만 다루고, 상세는 14·15·21·22·23·24(주관식 들어간 문항)에 집중한다.

## 1️⃣ 캔버스 채점 루브릭 (내용의 뼈대 — 출처: ccfm-wiki + 변하정/신동협 채점 실측)

각 칸은 **요구하는 추상도**가 다르다. 기능 스펙을 모든 칸에 복붙하는 게 **최대 감점 패턴**. 채점은 "칸이 요구하는 레벨까지 내려갔는가"로 본다.

| 문항 | 무엇을 보는가 | ✔ 잘한 신호 | ✕ 감점 신호 | › 처방 |
|---|---|---|---|---|
| **14 · 본질 (문제→해결)** | Before/After가 **고객 욕망·감정**까지 내려갔나 | After가 감정·자기인식("거울 안 보는 자신감")까지 도달 | After가 **기능 차원**(「47h 고정」)에서 멈춤 / Before가 기능적 묘사 | 기능 뒤에 "그래서 고객은 뭘 느끼나?"를 3번 묻기 (47h→종일 또렷→자신감) |
| **15 · 근거 캔버스** | 숫자·사회적·통념·컨셉 칸을 **정의대로** 채웠나 | 구체 숫자(162.02% 등) · 컨셉=유일 한 단어 · 통념 깔고 반전 | **칸 정의 혼동**(사회적근거 칸에 타깃/권위를 적음) | 사회적근거=판매량·재구매·후기(숫자) / 컨셉=최초·최고·유일 한 단어 / 본질=고객 욕망 |
| **21 · 증거·권위 테스트** | 싸게 만들고 테스트해 **승자만 실비용** 투입하는 흐름 | ①가설 소재 AB → ②**선행지표(CTR·CPC)** → ③**후행지표(CVR·구매)** 승자선별 → ④실비용(임상·체험단·시딩) | 측정/선별 단계 뒤섞임 · 4단계가 "돈 들지만 영구 자산"임을 안 짚음 | 2단계(측정)와 3단계(승자만 남기고 끄기)를 분리 |
| **22 · 허브 키워드** | 효능을 **상상하게 만드는 조어**를 만들고 **자사로 수렴**했나 | 멜라오프·끌올라겐 같은 조어 창의성 | **예시 카테고리만 풀고 자사(제품) 미적용** | 예시 풀었으면 "→ 그래서 우리 OO" 자사 허브키워드까지 만들기 |
| **23 · 유명인·권위·배경** | 인물·장소 선정 + **"왜 이 권위가 제품 증거인가"** | 인물/배경 합리적 + 증거화 칸 채움(가혹환경→그래도 유지) | "왜 증거인가" 누락 / 표면적 연결 | 무대=땀·조명·장시간 가혹환경 → "거기서도 안 무너짐"을 B&A 증거컷으로 |
| **24 · 키워드 마인드맵** | 막연→구체→부위→강한페인→시술 **연쇄**, 끝은 **자사 브릿지** | 동어반복 없이 단계 상승 | 1→2 **동어반복** / 종착이 경쟁시술에서 끝나고 자사 브릿지 누락 | 끝을 "시술 대신 매일 5초 OO"로 자사 제품에 착지 |

**공통 감점(대개 7할)**: ① 기능 스펙을 본질·사회적근거·컨셉 칸에 복붙 → "한 단계 더 내려가기" 안 함. ② 허브·마인드맵·유명인을 **예시만 풀고 자사로 수렴 안 함**. 이 둘만 고치면 통상 +10점.

채점 태도: **잘한 건 구체적으로(숫자·문구 인용해서) 칭찬**, 못한 건 **감점 사유 + 모범 리라이트**까지. AI가 채점하는 C 점수의 근거를 사람이 납득하게 만드는 게 목적.

## 2️⃣ 생성 절차

### STEP 1 — 답안 채점
입력의 14·15·21·22·23·24 답안을 루브릭으로 채점. 각 문항당 ✔/✕/› 라인 2~4개 도출. 추가로:
- **공통 강점 1줄 / 공통 약점(감점의 7할) 1줄** (배너 2개)
- **다음 진단 전 연습 3가지** (가장 임팩트 큰 처방을 문항 ref와 함께)
- **잘한 것 푸터** (모범 영역 나열 + "한 단계만 붙이면 N점대" 마무리)
- 14번엔 **모범 리라이트 quote** 1개 권장.

### STEP 2 — 템플릿 채우기
`assets/template.html` 복사 → 플레이스홀더 교체. **`<style>` 와 클래스 구조는 손대지 않는다.**

| 플레이스홀더 | 내용 |
|---|---|
| `{{TITLE}}` | `응시자명 · 퍼포먼스 캔버스 진단 피드백` |
| `{{NAME}}` `{{DATE}}` `{{PRODUCT}}` | 응시자명 / YYYY-MM-DD / 제품명 |
| `{{SUBTITLE}}` | `주관식(C) 집중 분석 — 본질 / 근거 캔버스 / 테스트 / 허브 키워드 / 유명인 / 마인드맵` |
| `{{SCORES}}` | 점수 카드 4개 (아래 패턴). **종합과 C는 `sc hot`**(핑크 강조), A·B는 일반 |
| `{{BANNERS}}` | 강점 배너(`banner`) + 약점 배너(`banner weak`) |
| `{{CHIPS}}` | 6개 문항칩 (14·15·21·22·23·24) |
| `{{PRACTICE}}` | 연습 3행 |
| `{{DETAIL}}` | 문항별 상세 블록 6개 |
| `{{FOOTER}}` | 잘한 것 1문단 |

### STEP 3 — PDF 변환 + 1페이지 핏
```powershell
powershell -ExecutionPolicy Bypass -File "<skill>/assets/build_pdf.ps1" -Html "<출력.html>"
```
- Edge headless 로 인쇄, pymupdf 로 페이지 수 검사, **1페이지 될 때까지 zoom 자동 하향**.
- 스크립트 못 쓰면 수동: Edge `--headless --print-to-pdf=... --no-pdf-header-footer "file:///..."` → `python -c "import fitz;print(len(fitz.open('x.pdf')))"` 로 확인 → 2페이지면 `<style>` 의 `zoom:.71` 을 `.69→.67...` 로 내려 재변환.
- **검증 필수**: 마지막에 PDF를 이미지로 렌더해 색(검정·흰·핑크)·1페이지를 눈으로 확인.
  `python -c "import fitz;d=fitz.open('x.pdf');d[0].get_pixmap(matrix=fitz.Matrix(1.6,1.6)).save('_chk.png')"` → Read 로 보기 → 확인 후 `_chk.png` 삭제.

### STEP 4 — 출력
`<응시자>_퍼포먼스캔버스_피드백.html` + `.pdf` 두 파일. 경로와 채점 요약(종합/강점/핵심 보완 2가지)을 사용자에게 보고.

## 3️⃣ 컴포넌트 HTML 패턴 (그대로 복사해 값만 교체)

**점수 카드** (`{{SCORES}}` — 4개. 종합·C는 `hot`, 강조 tag는 `pk`):
```html
<div class="sc hot"><div class="lab">종합</div><div class="big">85<small>/100</small></div><div class="tag">70점 · 액션러닝</div></div>
<div class="sc"><div class="lab">A 랜딩 OX</div><div class="big">11<small>/13</small></div><div class="tag">85% · 안목 양호</div></div>
<div class="sc"><div class="lab">B 본질 객관식</div><div class="big">23.6</div><div class="tag pk">95% · 프레임 우수</div></div>
<div class="sc hot"><div class="lab">C 주관식(AI)</div><div class="big">17.4</div><div class="tag pk">76% · 디테일 강함 ◀</div></div>
```

**배너** (`{{BANNERS}}`):
```html
<div class="banner"><span class="h">강점:</span> 숫자 근거 구체성 · 테스트 <span class="pk">선행/후행 4단계</span> 분리 … 틀이 단단함.</div>
<div class="banner weak"><span class="h">보완 (감점의 7할):</span> ① <b>본질</b>이 기능 차원(<span class="pk">「47h 고정」</span>)에서 멈춤 … ② 허브·마인드맵 <b>자사 미수렴</b> — <span class="pk">"→ 그래서 우리 OO"</span> 브릿지만 붙이면 <b>85→90점대.</b></div>
```

**문항칩** (`{{CHIPS}}` — 6개):
```html
<div class="chip"><span class="pill">14 · 본질</span><span class="txt"><b>문제→해결</b> · 기능≠본질, After가 「47h」에서 멈춤 <span class="pk">★최대 보완</span></span></div>
```

**연습 행** (`{{PRACTICE}}` — 3개. `<div class="n"></div>`는 자동 번호):
```html
<div class="row"><div class="n"></div><div class="c"><b>레벨 올리기</b> — 기능 뒤 "그래서 고객은 뭘 느끼나?" 3번 묻기 <span class="ref">→ 14번</span></div></div>
```

**문항 상세 블록** (`{{DETAIL}}` — 6개. badge는 보완=기본(핑크)·잘함=`good`(민트). 마크: `ok`✔민트 / `no`✕핑크 / `note`›회색):
```html
<div class="qblock">
  <div class="qhead">14 · 본질 문제 → 해결 <span class="badge">★ 최대 보완 · 기능≠본질</span></div>
  <div class="qbody">
    <div class="ln"><div class="mk no">✕</div><div class="body"><b>Before</b> "…" = <span class="pk">기능적 묘사</span>. 감정 페인까지 내려야 함.</div></div>
    <div class="ln"><div class="mk note">›</div><div class="body"><b>After</b> "…" — 진전. <span class="pk">한 단계 더</span>: 또렷 → "눈화장을 잊고 사는 자신감".</div></div>
    <div class="ln"><div class="mk ok">✔</div><div class="body">근거(47h)를 쥐고 있어 리라이트가 쉬움.</div></div>
    <div class="quote"><b>모범 리라이트</b> — Before: … / After: … <i>(근거 47h)</i></div>
  </div>
</div>
```

## 4️⃣ 양식 불변식 (어기면 "양식 안 봤다"는 지적의 원인 — 반드시 준수)

- 배경은 **검정**(`#0a0a0d`), 글씨 **흰색**, 강조 **핑크 `#ff2d78`**, 체크(✔)는 **민트 `#4edfa0`**. 다른 테마(흰 배경 등)로 만들지 말 것.
- 점수 카드는 4개, **종합·C 카드만 핑크 테두리(`hot`)**.
- 결과는 **A4 세로 1페이지**. 넘치면 내용을 줄이지 말고 먼저 `zoom` 으로 핏(하한 .60).
- 칭찬/지적에는 **응시자가 실제로 쓴 문구·숫자를 인용**. 일반론 금지.
- 만들기 전 원본 양식(변하정/신동협 PDF)이 있으면 **이미지로 렌더해 색을 눈으로 확인**하고 시작(텍스트만 추출하고 색 추정 금지 — 과거 실패 사례).

## 의존성
- Edge(headless 인쇄), Python+`pymupdf`(페이지 검증·렌더). `pip install pymupdf` 1회.
- 없으면 HTML만 내고 "브라우저에서 인쇄→PDF, 배경 그래픽 켜기" 안내.
