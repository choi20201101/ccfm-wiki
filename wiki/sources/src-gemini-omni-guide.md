# src-gemini-omni-guide

> Google Flow 안에서 동작하는 멀티모달 영상 생성 모델 **Gemini Omni (Omni Flash)** 사용 가이드.
> Gemini의 추론 + DeepMind의 **Nano Banana(이미지) + Veo(영상) + Genie(월드 시뮬레이션)** 결합.
> 영상의 다음 장면을 예측하고 물리(중력·운동·유체)를 이전 모델보다 더 사실적으로 시뮬레이션.

## 한 줄 포지셔닝
**Seedance** = "0 → 1 생성"에 강함 (새 영상 만들기).
**Omni** = "기존 영상 수정·변형"에 강함 (리믹스/합성/리얼리스틱 변환).

즉 t2v 첫 컷은 시댄스, 이후 변형·합성·일관성 유지는 옴니가 더 유리.

---

## 준비물 / 비용
- **계정**: Flow Ultra
- **모델 선택**: Flow Veo → 모델을 `omni flash`로 전환
- **차감 크레딧**
  - **v2v** (영상→영상 변형) : 40 크레딧
  - **t2v** (텍스트→영상) : 15 / 20 / 25 / 30 크레딧 (길이/품질)

## 진입 경로
1. Flow Veo 진입 → 모델 드롭다운에서 **`omni flash`** 선택 → 원하는 기능 사용
2. **에이전트 모드**: 프롬프트 작성칸 하단 **에이전트** 클릭 → 우상단 펼침 버튼 → 우측 대화창에서 자연어로 지시

## 추가된 주요 기능 (8가지)
| 기능 | 설명 |
|------|------|
| **Video → Video** | 기존 영상을 리믹스/변형 |
| **Mixed Input** | text + image + audio + video 어떤 조합이든 섞어 하나의 일관된 영상 출력 |
| **복잡한 아이디어 시각화** | 짧은 프롬프트로도 복잡 개념을 시각 자료 + 설명 영상으로 |
| **물리 표현** | 중력·운동에너지·유체역학을 사실적·자연스럽게 |
| **디지털 아바타 + 음성 합성** | 사용자 음성 → 아바타 립싱크 (UGC 워크플로 직결) |
| **액션 재상상 (Reimagine)** | 찍어둔 영상의 **행동 자체**를 변경 요청 |
| **텍스트 렌더링 (다국어)** | 영어 손글씨, 중국어·일본어·**한국어** 렌더링 |
| **Audio → Video** | 음성/사운드 입력 → 분위기에 맞는 영상 (예: 음악 ↔ 비주얼) |

## 주요 장점
- Veo UI 안에서 바로 사용 → 접근성 우위
- 입력 조합·변형 범위가 넓어 자유도 高
- **영상 + 누끼 이미지 1스텝 합성** (기존 2스텝: 이미지 합성 → I2V 를 1스텝으로 단축)
- 캐릭터 에셋으로 **일관성 유지**

## 활용 워크플로 9종 (실전)

### 1. 영상 내 이미지 합성 (제품/객체 교체)
**입력**: 베이스 영상 + 누끼 이미지 (제품/객체)
**프롬프트**: `영상 속 [부위]가 @[파일명] [객체]를 들고 있도록`
**변형**: `영상 속 [제품]을 @[파일명] [객체] 로 변경`

대표 케이스 (메라블 루비알엔 손-제품 합성, 슬리퍼→신발 교체).

### 2. 스톡소스 생성
3D 인포그래픽 모션 영상 등 t2v 베이스 소스 생성. 토글 안에 프롬프트 보관.

### 3. 영상 내 특정 행동 변경 (Reimagine)
**프롬프트**: `영상 내 [기존 행동]을 [새 행동]으로 변경`
예: 따봉 → 브이 사인.

### 4. 모델 목소리 변경
**프롬프트**: `한국인 [성별] '[대사]'라고 말하는 장면`
1번/2번 목소리 옵션 비교 가능.

### 5. 영상 내 배경 변경
**프롬프트**: `인물은 변형 없이 배경만 [상세 묘사]로 변경`
인물 동작·외모 보존 + 환경만 교체.

### 6. 영상 내 인물 복장 및 외모 변경
입력 영상 + 누끼 의상 이미지 → 인물 복장 교체. 인터뷰 영상 등에 활용.

### 7. 앵글 및 구도 변경
**프롬프트**: `[하이앵글/로우앵글/백뷰/오버숄더]로 변경`
같은 장면을 다른 카메라 워크로 재생성.

### 8. 캐릭터 기능 (일관성 유지)
1. 프로젝트 생성 → **캐릭터** 항목 진입
2. 프롬프트로 새 캐릭터 생성 OR 캐릭터 에셋 첨부
3. 성격·음성 (선택) 지정 → 캐릭터 에셋 완성
4. 프롬프트 박스 좌측 하단 **+** 로 캐릭터 에셋 지정
5. `@에셋명` 으로 태그하여 프롬프트 작성

→ 카페에서 두 여성 대화 / 한강 러닝 등 멀티-씬에서 동일 캐릭터 유지 가능.

### 9. 실무 적용 (착한구두 럭셔리 패션 에디토리얼 케이스)
- **t2v**로 기초 워킹 영상 출력
- **영상 + 이미지 합성**으로 워킹 영상 내 샌들 이미지 + 복장 묘사 프롬프트 추가
- **프리미어 컷편집 + 디졸브**로 자연스러운 컷 전환

결과물: Korean woman shoe fashion film, linen pants, minimalist outfit, oversized blazer 등 5개 변주.

## 활용 포인트 (CCFM 관점)

### 광고 소재 파이프라인 적용
- **누끼 합성 1스텝화** → 기존 [[da-creative]] 워크플로(이미지 합성 → I2V 2스텝)를 1스텝으로 압축
- **캐릭터 에셋 일관성** → 시리즈 광고/B&A 영상에서 동일 인물 유지에 유리
- **앵글 변경** → 같은 컷에서 하이/로우/백뷰 변주 자동 생성 → A/B 테스트 다양성 확보
- **배경 변경** → 실내·실외·계절 변주 = 동일 인물·동일 동작 + 환경만 다른 시리즈 가능

### Seedance와 분업
| 단계 | 도구 | 이유 |
|------|------|------|
| t2v 첫 컷 생성 | Seedance | 0→1 생성 강점 |
| 합성·변형·일관성 | Omni Flash | v2v + 이미지 합성 + 캐릭터 에셋 |
| 컷편집·디졸브 | Premiere | 영상 결합 |

### 한계 회피 패턴
- 한국어 텍스트 합성은 Omni 단계에서 시도 X → 후처리 단계에서 처리
- 스토리보드 → 영상 변환은 프롬프트 한 줄 더 명시적으로 (`스토리보드 기반 [브랜드] [제품] [포맷] 영상 제작`)
