# 실패 교훈 (Lessons Learned)

## [2026-04-12] LaMa가 항상 OpenCV보다 좋은 건 아니다

**상황:** Gemini 생성 이미지 로고 제거 기술 비교 테스트
**결과:** simple-lama-inpainting < OpenCV TELEA+NS 블렌딩

**이유:**
- LaMa는 넓고 복잡한 텍스처 복원에 최적화 (벽돌, 풀밭 등)
- 극소 마스크(25×31px) + 단순 배경에서는 오버스펙 → 번짐 발생
- pip 패키지(simple-lama-inpainting)는 원본 LaMa의 70~80% 성능

**적용 원칙:**
> 마스크 영역이 이미지 대비 5% 이하 + 주변 배경 단순 → OpenCV 먼저 시도
> LaMa는 복잡 배경 or 넓은 영역(10%+) 제거 시 사용

## [2026-04-13] i-boss 201건 실패 교훈 (소스: [[wiki/sources/src-iboss-choi-jaemyeong.md]])

### 플랫폼·매체 탓 금지
- 관찰: "페북이 망했다?" 아니, 제품이 망한 것. 플랫폼 탓은 하수
- 조건: 저효율 발견 시 제품·콘텐츠·타겟 먼저 점검
- confidence: high

### 전단지 대행 첫회만 효과
- 관찰: 전단지 배포 대행은 첫 회차만, 2회차부터 급격히 감소
- 대안: 직접 우편함 배포가 더 효과
- 필수: "보고 왔음" 입증할 특정 혜택 명시 (측정 위해)

### 홍보물 과다 배치는 역효과
- 소비자가 광고로 인식해 무시 → 1 메시지 집중 원칙
- 성공 공식: 고기 사진 크게 + 사람 얼굴 + 텍스트 최소 + 기대 카피

### 인도네시아 크로스보더 실패 (CCFM 과거 기록)
- 배송 7일 → 반품률 40%
- 대안: 현지 창고·플랫폼 직입점

### AI 70% 벽
- 바이브 코딩은 70%까지만, 나머지 30%·에러 대응 위해 기본 개발 개념 필수
- "알아서 해줄 거야" 하면 자동화 70%에서 멈춤
- 암묵지·기준 명시화가 95%+ 자동화 조건

### 단일 LLM 의존 금지
- Gemini·ChatGPT·Claude·Perplexity 장단점 상호보완 필요
- "글쓰기는 Claude, 코딩은 ChatGPT" 최소 2~3개 동시 활용

### 타겟 집착 금지 (맞춤타겟 만능 아님)
- 맞춤/유사 타겟으로 모수 축소해 빈도 올리는 전략은 구매로 이어지지 않을 수 있음
- 머신러닝은 '구매가능성'보다 '클릭확률 높은 고객'에 노출 편향

### 데이터쇼핑 맹신 금지
- 자사몰 데이터는 결과일 뿐 — 검색량 트렌드 같은 "의도 데이터"를 먼저
- "데이터에서 답을 찾지 말고 원하는 행동을 설계한 뒤 측정"

### UTM ROAS 단독 판단 금지
- UTM은 마지막 클릭 — 간접 전환·전체 여정 누락
- 4~5개 매체 믹스 시 왜곡 심화

### 유행편승의 함정
- "골프 전용" 포지셔닝은 유행 종료 후 급락
- 단기 폭발보다 근본 USP 구축이 지속 가능

### 매출 ≠ 이익
- 대표 함정: 재고·세금·마진 착각
- 광고비는 원가·고정비·변동비·세금 뺀 순이익 기반 책정

### "1등 모방"은 실패 지름길
- 대안: 1등의 문제점을 찾아 보완

### 전환 캠페인 ≠ 성과형 DA
- 성과형 DA는 트래픽 × 사이트컨디션 투트랙
- 구매완료 대신 조회만 측정되는 경우 주의

### Performance Max 저예산 실패
- 일 10만원 이상 예산 권장, 저예산은 학습 불안정

### 파워컨텐츠 전환 부적합
- 인지도엔 좋지만 전환 부적합 — 광고 인식 이탈률 높음

### 0에서 창작은 리스크
- 성공 사례 모방해 빠르게 창조하는 것이 현실적
- 0.1% 성공신화 탈피 — 제품·고객·시즌·재구매 4요소 조화
