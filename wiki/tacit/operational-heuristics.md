# 운영 노하우 (최재명 대표)

> 출처: i-boss 201건. 소스: [[wiki/sources/src-iboss-choi-jaemyeong.md]]

## 퍼포먼스 광고 운영

### 광고 구조 3단
- 캠페인(목적) → 광고그룹(타겟·지면·입찰) → 소재
- 캠페인 목표 4가지: 전환·방문·도달·조회
- 실제 운영 세팅 핵심은 광고그룹

### 빈도 2~4회가 최적
- 메타 광고 최적 빈도 2~4회, 높다고 매출 비례 안 함
- "여러 번 = 구매"는 미신
- 예산 적으면 빈도 늘리기보다 3회 내 전환시키는 콘텐츠에 집중

### 리타게팅 5% 이내
- 단순 방문자 리타게팅은 의도 혼재 → 비효율
- 행동 기반 모수(장바구니·회원가입·결제시작)만 리타겟
- 신규 고객 확보에 집중하면 머신러닝이 자동 최적화

### 소재 수 제한
- 1그룹 최대 10소재 테스트, 부진은 즉시 OFF
- 그룹 10개+는 무의미, 10개 내외 시작 → 3~4개로 축소
- 좋은 소재는 3주~6개월 유지

### ABO → CBO 이원 운용
- 신규 계정은 ABO에서 검증 → CBO로 확장

## 전환·측정

### 측정할 수 없으면 개선 없음
- 쿠키·AD ID·UTM 3가지 추적 방식 정확도·적용 범위 차이 이해
- 직접전환(클릭 직후) vs 간접전환(며칠 후) 구분
- 페북 매니저·GA 여러 도구 교차 검증으로 오차 최소화

### 1일 기준 + 실시간 대시보드
- 광고비/매출/CPC/CTR 즉각 ON·OFF
- 연령·성별·시간대 매출 분석으로 '감→데이터' 전환

### 전환액션 우선순위
- 구매완료 > 외부결제 > 결제시작 > 회원가입 > 장바구니
- 초기 50개 전환으로 학습 완료

## CRM · LTV

### 재구매 > 신규
- 기존 고객 CPS 수준 할인/쿠폰이 신규 광고비보다 ROI 우수
- 멤버십 레벨링(LV1~Master) 게임화
- 순이익 20% 내에서 포인트·쿠폰 부여

### 플러스친구 주 3회 발송
- 회당 5~7천만 매출 상승 ("아끼면 똥")
- 채널톡·데이터라이즈로 구매빈도·생일·가입일 자동 CRM

### 카톡채널 1년 1,000명
- 수수료 없는 직접 채널, 유지율 높음
- 배후세대 재개발로 소실된 오프라인 매출 보완

## 물류·발주·공급

### 폭발 매출보다 점진 확대
- 발주 타이밍 맞춰 1~2개월 점진 확대
- 광고비는 원가·고정비·변동비·세금 뺀 순이익 기반 책정

### 고정비 ≠ 변동비
- 고정비는 줄이기 어렵고 변동비는 대량생산으로 단가 절감
- 반품·환불·배송에서 고객 배려가 악성 리뷰 방지

### 쿠팡 로켓그로스 전환 시 최소 2주 재고 확보 (CCFM 내부 암묵지)

## 랜딩·UX

### 사이트 컨디션 50%+
- 전환율 50%+가 사이트 컨디션 — 기본기 없는 광고비는 낭비
- 부정 리뷰 상단 노출 → 효율 50%+ 악화

### 사소한 불편함 개선이 CVR 열쇠
- 리뷰 보는 중에도 구매 버튼 활성화
- 회원가입 후 원래 페이지 복귀
- 옵션별 리뷰 분리
- 카카오싱크 간편가입 + 1분 유휴 사용자 푸시 리타겟팅

### 상세페이지는 '투머치'하게
- GIF/비포애프터로 이득 즉각 인지
- 광고와 내부콘텐츠 소구점 일치 필수

## AI 자동화 운영

### 20분 → 2분 과제부터
- "사람이 꼭 안 해도 되는 일을 사람이 하는지" 진단이 출발점
- n8n + Claude 조합으로 이미지 리사이즈·경쟁사 모니터링 자동화
- 월 10~20만원 투자로 해결

### AI 도입 5단계 로드맵
- L1(재미) → L5(에이전트화)
- L3~L4면 리소스 50% 절감 가능 (월 8천만원 사례)

### 정량은 자동화, 창의·판단은 인간
- 유튜브 정책 변화가 시사: "대량 자동화 시대 종료"
- AX팀 신설로 전사 반복업무 전담

### 프롬프트는 LLM에게 짜게 하기
- "반복은 AI, 판단은 사람"
- 3~5회 수정 루프로 퀄리티 확보

### 단계별로 쪼개기
- 일 한꺼번에 많이 시키면 AI 집중력 분산
- 단계 분할 시 100% 집중

## 대행사·조직 운영

### "사람이 곧 상품"
- 직원 성장이 대행사 경쟁력 본질 — 시스템으로 사람 키우기
- 재무·고객·학습성장·프로세스 4관점 BSC

### 대표 의존 탈피
- 레벨별 성장 로드맵·가이드 체계화
- 누구나 전문가 수준 도달 가능한 구조

### 대행사·클라 상생
- 광고 대행사는 '실험사', 클라 가감없는 협력이 성과 2배

### [2026-04-28] 브랜드 시너지 3 시나리오 매트릭스 (퍼포먼스 마케팅)

기존 브랜드 안에 신규 카테고리 라인 런칭 가부 판단 룰. (간호사신발×착한구두 케이스에서 추출)

**판정 기준** (모브랜드 가격 M, 신규 카테고리 권장가 N):
- **N/M ≤ 1.2x** → 시나리오 A (메인 라인 직접 출시) — 단기 ROAS 2.5~3.0x 최고지만 시장 1/3만 점유
- **1.2x < N/M ≤ 2.5x** → 시나리오 B (서브브랜드) ★ — ROAS 2.0~2.5x, 화이트스페이스 점유, CAC -40%
- **N/M > 2.5x** → 시나리오 C (완전 신규/자회사) — ROAS 1.2~1.5x, BEP 24개월+

**시나리오 B 절대 양보 불가 3 조건**:
1. 카테고리 권위 인증 (의료=KFDA, 식품=HACCP, 헬스=식약처) — 12-18개월 신규 진행
2. 임상/사용 검증 그룹 (200명+) — 모브랜드 자산만으로는 신뢰 부족
3. 서브 인스타·패키징·자사몰 카테고리 분리 — 정체성 충돌 격리

이 3개 빠지면 시나리오 A로 추락, 있으면 신규 단독(C) 효과를 70% 가격에 얻음.

**관찰**: 시너지 PRO 7축 / CON 6축 평가표 사용. 각 ★ 가중치, ★★★가 3개 이상 PRO면 진입 권장.
**조건**: B2C 브랜드 확장 (마케팅 인프라 공유 가능), 신규 카테고리가 기존과 *이종*일 때.
**출처**: nurse/reports/11_chaakan_synergy.md
**confidence**: low (1회, 본 케이스에서 도출 — 추가 케이스 검증 필요)

### [2026-05-12] NAS 큐 패턴 — 단계 간 인터페이스를 폴더 존재 여부로 표현
**룰**: 멀티스텝 자동화 파이프라인을 만들 때 단계 간 결합은 함수 호출이 아니라 **NAS 폴더 + 폴더명 규칙(`YYYYMMDD_브랜드[_N]`) + done/ 이동**으로 표현한다.

**Why**: 3팀 영상 자동화 M1~M5가 실제로 이 패턴으로 운영중. 단계별 스킬이 서로 코드 의존 없이도 5단 파이프라인이 굴러감. 어느 단계가 실패해도 NAS의 폴더 상태를 보고 그 자리에서 재실행 가능. 새 단계 추가도 input/output 폴더 규약만 맞으면 됨.

**How to apply**:
- 단계 N+1은 단계 N의 출력 폴더를 ls해서 미처리 후보를 찾는다 (already-in-done 이면 skip)
- 처리 완료 후 입력을 `done/`으로 이동 (SMB에서는 `cp -r` + `rm -rf`)
- 단계 간 데이터는 파일로만 주고받음 (함수 반환값 X) — `scenes.json` 같은 단일 스키마 1개로 합의
- 사람이 끼어드는 게이트(M5a의 "ref 크로스체크 결과 확인" 같은)는 비싼 작업(M5a/c) 전에 둔다 — 무한 자동화 금지

**confidence**: high (5단 × 다중 브랜드 운영 검증)

### [2026-05-08] 사용자 Chrome 프로세스 전역 종료 금지

**룰**: `Get-Process chrome | Stop-Process -Force` · `taskkill /IM chrome.exe` · `pkill chrome` 같은 전역 Chrome 종료 명령 절대 금지. 동등한 `chromium` / `chromedriver` 전역 종료도 동일. node·python·ffmpeg 같은 다른 광범위 프로세스도 사용자 확인 없이 일괄 kill 금지.

**Why**: 2026-05-08 네이버 블로그 스크래퍼 디버깅 중 DrissionPage가 좀비 Chrome 잡으려는 줄 알고 모든 Chrome 프로세스를 한 번에 죽였더니 **사용자가 동시에 돌리던 다른 자동화 Chrome들도 다 같이 죽음**. 사용자는 항상 여러 Chrome 자동화를 병렬로 운용 중 — DrissionPage / Selenium / 사용자 수동 세션 등.

**How to apply**:
- DrissionPage 인스턴스 충돌 시: chrome 죽이지 말고 **격리된 user_data_dir + 고유 포트**로 새 인스턴스 띄우기
  ```python
  co.set_user_data_path(tempfile.mkdtemp(...))
  co.set_local_port(9333)
  ```
- 추가 인자: `--no-first-run`, `--no-default-browser-check` (첫 실행 모달 회피)
- 정 chrome 정리가 필요하면 PID 명시해서 그 인스턴스만 종료

**confidence**: high (실제 사고)

### [2026-05-XX] User Temp 정리 background 실행이 자기 task output 삭제

**룰**: User Temp(`$env:TEMP` = `C:\Users\<user>\AppData\Local\Temp`) 정리 명령은 가급적 **inline** 실행. `run_in_background`로 돌리면 그 task의 output 파일(`%TEMP%\claude\...\tasks\<id>.output`)이 같은 Temp 폴더 안이라 cleanup이 자기 output을 같이 쓸어버려서 결과를 못 읽는다.

**Why**: Claude Code background task output 디렉토리는 `%TEMP%\claude\...` 경로 자동 생성. Temp 정리 명령이 자기 자식 디렉토리까지 sweep하면서 output도 같이 날아감. task-notification은 와도 Read로 결과 못 읽음.

**How to apply**:
- 가급적 inline 실행 (run_in_background 없이)
- 굳이 background로 돌려야 하면 cleanup 스크립트가 `$env:TEMP\claude` 폴더는 명시적으로 제외:
  ```powershell
  Get-ChildItem $env:TEMP -Exclude 'claude' -Force | Remove-Item -Recurse -Force
  ```
- 또는 output을 D 드라이브 등 다른 위치로 redirect
- output 없이도 검증 가능하도록 작업 끝에 디스크 여유 공간을 별도 명령으로 확인하는 follow-up 준비

**confidence**: high (재현 가능)

### [2026-05-29] AI 인용은 한 번 학습되면 다음엔 안 가져갈 수도 있다
- 관찰: LLM은 학습(RAG) 1차 → 웹서치 보완. 봇 크롤 = 인용·학습이지만, 한 번 학습·인용되면 재방문 안 할 수 있음.
- 조건: GEO 측정 시 — 봇 유입량(예: 누적 544만 중 절반이 봇)만으로 "노출 잘 됨" 판단하면 안 됨.
- 출처: 발화자 경험
- confidence: medium

### [2026-05-29] 자동화 콘텐츠는 인용 안 됨 — 사람 글+실제 사진과 이원화
- 관찰: 자동화로 찍어낸 글은 AI 글로 인식돼 걸러짐(인용 부적격). 실제 사람 글 + 직접 찍은 사진이라야 인용 적격.
- 조건: 발행 자동화 설계 시 — 자동화는 롱테일 키워드 커버리지·발행 효율까지만, 인용 노리는 핵심 콘텐츠는 사람이 직접.
- 함정: 외주 작가가 다시 LLM으로 쓸 수 있음 → 신뢰 가능한 사람 / AI 글 필터 필요.
- 출처: 발화자 경험
- confidence: medium

### [2026-05-29] 구글 어뷰징 = 도메인 영구 사망 → 정확성이 더 빠르고 안전
- 관찰: 구글은 어뷰징 판단 시 도메인을 영구적으로 죽임(복구 불가). 무리한 물량으로 밀면 도메인 통째 상실.
- 조건: 어뷰징 한계 실험은 버려도 되는 테스트 사이트(내구 테스트)로만. 메인 자산으로 실험 금지.
- 출처: 발화자 경험
- confidence: medium

### [2026-05-29] GEO 측정은 API·스크린 모드로 (내 PC 검색 금지)
- 관찰: 결과는 로그인·개인화로 사람마다 다름. 광고주 본인 PC로 검색하면 본인 사이트 안 뜸. API·스크린 모드로 지문 보내 나오는 값을 기준으로.
- 조건: "안 보인다"가 곧 "실패"는 아님 — 압도적으로 좋지 않으면 노출 자체가 안 됨.
- 출처: 발화자 경험
- confidence: medium

### [2026-05-29] 키워드는 문장형 롱테일을 폭으로 (메인 한 방 ✕)
- 관찰: 이제 단어가 아닌 문장형 질문 검색. 세부까지 다 잡으면 100만 규모, 각각 월 1회 노출돼도 누적 유입 큼. 메인 키워드는 경쟁 세고 전환 약함.
- 조건: 발행 자동화 = 문장형 세부 키워드 대량 수집 → 키워드별 콘텐츠 월 단위 지속 발행.
- 출처: 발화자 경험
- confidence: medium
