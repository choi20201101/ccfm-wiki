# 06. 실전 실패 & 교훈

> 33편 제작하며 겪은 함정과 해결 방법. 다음 프로젝트에서 같은 실수 반복 금지.

## 1. ❌ Kling 직통 API 계정 활성화

**증상**: 5개 Access Key × 2~3개 계정 모두 `code 1003 "Authorization is not active"`
**원인**: 리소스 패키지가 살아있어도 **API 접근 권한이 별도 계정 레벨에서 비활성**
**해결**:
- Kling CS에 `activate API access` 티켓 필수 (24~48h 소요)
- 임시 우회: **fal.ai 동일 모델 제공** — Kling Avatar, i2v, TTS 모두 프록시
**교훈**: 계약 전 API 호출 테스트 1회 의무화

## 2. ❌ 한국어를 Gemini 이미지에 baked-in

**증상**: "'속보'", "'300만원'", "'1일차'" 같은 한글 텍스트가 이미지에 포함되면 **깨진 글자 출력**
**원인**: Gemini/diffusion 모델은 한글 자모 조합을 정확히 렌더링 못 함
**해결**:
- 이미지 프롬프트에서 한국어 텍스트 **모두 영어 대체** ('BREAKING NEWS', '3 MILLION WON', 'DAY 1')
- 한국어 카피는 **ffmpeg/ASS 후처리 자막**으로만
- `Korean text`, `Korean numbers` 같은 qualifier까지 제거 필요 (2차 pass)
**교훈**: 비라틴 문자는 이미지 생성 AI에 넣지 말 것

## 3. ❌ 거울 셀피 페르소나 할루시네이션

**증상**: "욕실 거울 셀피" 시드 → **3명의 반사**가 동시 출현 (무한반사 효과)
**원인**: Gemini가 거울을 해석할 때 mirror-in-mirror 패턴으로 오버렌더
**해결**:
- 거울 관련 키워드 제거 → 평면 벽/침실 배경으로 대체
- 프롬프트 강제: `exactly ONE person in frame, NO mirror, NO reflections`
- Kling i2v `negative_prompt`에 `multiple people, mirror reflection`
**교훈**: 반사/투명/복잡 기하학 요소는 AI가 약함

## 4. ❌ 크림 통 "먹는 것처럼" 보이는 손동작

**증상**: "applying cream" 모션 프롬프트 → 크림 통이 얼굴로 가까이 오면서 먹는 장면 연출
**원인**: 손 근처 + 입 근처 모두 자연스러운 패턴이라 AI가 헷갈림
**해결**:
- 프롬프트: `jar stays at chest level (not near mouth)`
- B-roll 모션: `fingertip only, no applicator, scoop then apply on cheek (not lips)`
- Negative: `jar near mouth, cream in mouth, eating cream, licking product`
**교훈**: 물체-인체 상호작용은 명시적 위치 제약 필요

## 5. ❌ 이상한 애플리케이터 (Q-tip, 솜뭉치)

**증상**: "apply cream" 프롬프트 → 존재하지 않는 applicator wand 생성
**원인**: 화장품 적용 = 다양한 도구 사용 패턴이 AI 학습에 섞임
**해결**:
- `fingertip dabs directly on cheek` 명시
- Negative: `cotton swab, applicator wand, Q-tip near face`
**교훈**: 생활 도구의 AI 기대값 ≠ 실제 브랜드 가이드

## 6. ❌ 자막 EN 축약 vs 오디오 불일치

**증상**: 오디오는 전체 문장 `"POV: I walk into my derm office..."` 읊는데 자막 EN은 `"POV: me at derma"` (패러프레이즈)
**원인**: 서브큐로 쪼갤 때 EN을 의역·축약
**해결 (Option B)**:
- EN은 **오디오 실제 문장을 쉼표/마침표 경계로 분할**
- KO는 해당 EN 청크의 **의미 번역**
- 시간 = 씬 duration × (청크 char수 / 총 char수)
**교훈**: 서브큐에서도 EN은 오디오 그대로 유지

## 7. ❌ 중년 한국어 부적합 자막

**증상**: "소프트걸 불가능", "풀스택 미백", "ㅋㅋ", "고고" 같은 표현으로 40~60대 타겟 이해 불가
**원인**: 기계 번역 + Z세대 슬랭 그대로 사용
**해결**:
- 전체 한국어 자막 수동 교정 (표준어)
- 금지어 리스트 운영 (04_자막_가이드.md 참고)
- 타겟별 어휘 풀 분리
**교훈**: AI 번역은 세대별 어휘 분리 학습 안 돼 있음 → 수동 검수 필수

## 8. ❌ 플로팅 PIP가 자막을 가림

**증상**: 임상 PIP overlay가 하단 자막(y~1500)과 겹침
**원인**: PIP y좌표 1100으로 설정 (중간) → 크기 440px → 1540까지 뻗음
**해결**:
- PIP y좌표 540 (상단 카피 바로 아래, 아바타 상단)
- 크기 440 유지하면 580~980 영역 (자막 안 가림)
**교훈**: 오버레이 배치 전에 자막 영역 먼저 정하기 (safe zone)

## 9. ❌ 상단 카피 노란 stroke 촌스러움

**증상**: 노란색 배경 border로 PIP 테두리 → 저품질 느낌
**해결**:
- 검정 굵은 stroke (20px) + 반투명 외곽 (8px @ 0.35)
- pad 2중 패턴으로 drop shadow 근사 구현
**교훈**: 디자인 토큰(색/굵기)이 시각 신뢰도 직결

## 10. ❌ FFmpeg filter_complex 인덱스 하드코딩

**증상**: B-roll 4개 기대하고 `[5:v][6:v][7:v]` 하드코딩했는데 실제 3개 입력 → 필터 파싱 실패
**원인**: `cand_all` dedup으로 3개로 줄었는데 인덱스는 4개 기준
**해결**:
- 동적 계산: `n = len(broll_specs); sb_idx = n+1; sa_idx = n+2; ca_idx = n+3`
**교훈**: 파이프라인 필터 인덱스는 항상 동적

## 11. ❌ `force_original_aspect_ratio=cover`

**증상**: ffmpeg 필터 exit -22 에러
**원인**: `cover`는 유효 값 아님. `increase` 또는 `decrease`만 지원
**해결**: `scale=1080:1920:force_original_aspect_ratio=increase,crop=1080:1920`
**교훈**: ffmpeg 필터 옵션 항상 `ffmpeg -h filter=scale` 확인

## 12. ❌ 배치가 조용히 죽음 (`| head -5`)

**증상**: `python script.py | head -5` → 5줄 이후 SIGPIPE로 Python 프로세스 종료
**원인**: head가 stdout 닫으면 flush 시도하던 Python이 죽음
**해결**: pipe 없이 파일 redirect (`> log.txt 2>&1`)
**교훈**: 백그라운드 배치는 stdout/stderr 파일 redirect 원칙

## 13. ❌ Python subprocess.run `capture_output=True` 에러 숨김

**증상**: ffmpeg 실패 시 stderr 안 보여 디버깅 힘듦
**해결**: 에러 발생 시 `e.stderr.decode()[-300:]` 로그 출력
**교훈**: 실패 분기에 stderr 남기기 표준화

## 14. ❌ 백그라운드 프로세스 상태 오판

**증상**: task-notification "completed exit 0"이 왔지만 실제 영상 몇 개만 완성됨
**원인**: 프로세스는 정상 종료했지만 중간에 배치 loop 내부 예외 → 건너뜀
**해결**: 배치 마지막에 `DONE: X/Y` 명시 로그
**교훈**: 프로세스 exit와 논리 완료는 다른 차원

## 15. ❌ 첫 배치 자산 활용 실패

**증상**: 첫 배치 실패 → 재시작 시 모두 처음부터 재생성 (Avatar $4 × 20 = $80 낭비)
**해결**:
- 각 단계 idempotent하게 설계 (파일 존재하면 skip)
- 실패 지점부터 이어서 재개
**교훈**: 비싼 API 호출은 disk cache 필수

## 16. ⚠️ 레퍼런스 영상 복사본 1:1 재현 불가

**증상**: 카메라 앵글, 모션, 자막 리듬을 레퍼런스와 똑같이 맞추기 매우 어려움
**원인**: Avatar v2 Pro는 오디오 길이만 맞춰 줌 (세부 동작 제어 미약)
**해결**:
- 기획 단계에서 AI 한계 고려 (완벽 복제 X)
- 레퍼런스 "느낌"만 모방, 세부 창의 허용
**교훈**: AI 영상 = 레퍼런스 참고 + 변주 인정

## 💡 핵심 원칙 요약

1. **이미지 AI는 한글 X** → 텍스트는 후처리로
2. **얼굴 일관성 = 시드 reference** → 동일 identity 유지
3. **시뮬레이션 실패 시 disk cache** → 비용 절감
4. **자막 싱크 = 오디오 타임스탬프** → 청크 글자수 비례
5. **타겟 연령에 맞는 어휘 풀** → 수동 검수 단계 필수
6. **레이어 순서**: 배경 → B-roll → B&A → PIP → 자막 (위에서 아래 순 덮음)
