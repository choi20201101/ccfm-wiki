# M4 자동 편집 파이프라인 스킬

NAS에 배포된 M4 Python 모듈을 호출해 STEP1(오디오 컷) → STEP2(자막) → STEP3(XMEML) 빌드를 실행한다.
Python 모듈 자체는 NAS에 있으므로 별도 배포 불필요. 이 스킬은 호출 래퍼다.

---

## 운영 경로 (OS 자동 분기)

```
NAS (Mac):     /Volumes/01_마케팅사업부_직원/{NAS_USER}/3팀 영상 자동화/M4
NAS (Windows): \\192.168.0.103\01_마케팅사업부_직원\{NAS_USER}\3팀 영상 자동화\M4
```

| 변수 | 값 |
|---|---|
| `M4_BASE` | 위 NAS 경로 (OS에 맞게) |
| `M4_INPUT` | `{M4_BASE}/input` |
| `M4_OUTPUT` | `{M4_BASE}/output` |
| `PYTHON` | Mac=`python3`, Windows=`python` |

> 모든 처리는 NAS 경로에서 in-place 실행. 로컬 복사 없음.
> `m4_run.py`의 `BASE_DIR = Path(__file__).parent` 상대경로 기반이라 양쪽 동일 동작.

---

## Step 1. 호출 모드 결정

| 호출 형식 | 동작 |
|---|---|
| `/m4 브랜드명` | input/에서 해당 브랜드 폴더 1개 처리 |
| `/m4` | 감지된 프로젝트 목록만 출력 후 사용자 선택 대기 |
| `/m4 --all` | 미처리 프로젝트 전체 순차 처리 |

---

## Step 2. OS 판별 + 경로 설정

```bash
# Mac
M4_BASE="/Volumes/01_마케팅사업부_직원/{NAS_USER}/3팀 영상 자동화/M4"
PYTHON="python3"
```

```powershell
# Windows
$M4_BASE = "\\192.168.0.103\01_마케팅사업부_직원\{NAS_USER}\3팀 영상 자동화\M4"
$PYTHON  = "python"
```

NAS 도달 확인:
- Mac: `[ -d "$M4_BASE" ]`
- Windows: `Test-Path $M4_BASE`

도달 실패 시 "NAS 마운트/네트워크 확인 필요" 안내 후 종료.

---

## Step 3. 프로젝트 감지

`m4_run.py`가 자체적으로 `input/` 스캔 + 마커/TTS 페어링한다. 직접 호출:

```bash
# Mac
cd "$M4_BASE" && python3 m4_run.py --index 1   # 목록만 보고 멈추는 게 아니라 1번 실행
```

```powershell
# Windows
Set-Location $M4_BASE; python m4_run.py --index 1
```

`--index`만 호출해도 첫 줄에 감지된 전체 목록을 출력한다:
```
[M4] 감지된 프로젝트: N개
  [1] 20260427_자보티바_1  (MP3, 05_자보티바_M3_M4마커.json, 테마=네이티브)
  [2] ...
```

목록만 보려면 `--index 999` 같은 범위 밖 값을 넣어 출력 후 실패시키거나, 직접 `ls "$M4_INPUT"`로 확인.

---

## Step 4. 브랜드명 → index 매핑

`/m4 브랜드명` 호출 시:

1. `ls "$M4_INPUT"` (Mac) 또는 `Get-ChildItem $M4_INPUT` (Windows)로 폴더 목록 획득
2. `done`, `sfx`, `.DS_Store` 제외, `YYYYMMDD_브랜드[_N]` 패턴만
3. 입력 브랜드명이 폴더명에 포함되는 항목 필터
4. 여러 개면 사용자에게 선택 질문, 1개면 자동 진행
5. 매칭 폴더의 list 순서 = `m4_run.py` 감지 순서 (둘 다 sorted)
   → 매칭된 폴더의 알파벳 정렬 순위 = `--index` 값

⚠️ 안전한 방식: 직접 `--index`로 호출하지 말고 먼저 목록을 받아 정확히 매칭되는 번호를 확인 후 실행.

---

## Step 5. 실행

```bash
# Mac
cd "$M4_BASE"
python3 m4_run.py --index <N>
```

```powershell
# Windows
Set-Location $M4_BASE
python m4_run.py --index <N>
```

옵션:
- `--theme 공포|네이티브|정보성`: SFX 테마 자동 감지를 무시하고 강제 지정
- `--all`: 감지된 모든 프로젝트 순차 실행

실행 중 출력 단계:
```
[M4] STEP1 — 무음 컷편집
[M4] STEP2 — 자막 타이밍 추출      ← Whisper 첫 실행 시 모델 다운로드(~460MB)
[M4] STEP3 — XMEML 빌드
[M4] ✓ 완료 → output/YYYYMMDD_브랜드/
```

---

## Step 6. 결과 검증

`output/{날짜_브랜드}/` 폴더에 4개 파일 확인:

```
{프로젝트명}_tts_cut.mp3 (또는 .wav)
{프로젝트명}_subtitle_chunks.json
{프로젝트명}_subtitle_ko.srt
{프로젝트명}_timeline.xml          ← Premiere import 대상 (.xml, .fcpxml 아님)
```

```bash
# Mac
ls -la "$M4_OUTPUT/{날짜_브랜드}/"
```

```powershell
# Windows
Get-ChildItem "$M4_OUTPUT\{날짜_브랜드}\"
```

4개 모두 있고 `.xml` 파일 0바이트 아니면 성공.

---

## Step 7. done 이동 (선택)

M3 패턴과 동일하게 처리 완료 후 input 폴더를 done으로 이동.

```bash
# Mac
mkdir -p "$M4_INPUT/done"
mv "$M4_INPUT/{날짜_브랜드}" "$M4_INPUT/done/"
```

```powershell
# Windows (SMB mv 오류 방지: cp + rm)
New-Item -ItemType Directory -Path "$M4_INPUT\done" -Force | Out-Null
Copy-Item -Recurse "$M4_INPUT\{날짜_브랜드}" "$M4_INPUT\done\"
Remove-Item -Recurse -Force "$M4_INPUT\{날짜_브랜드}"
```

## 의존성 (최초 1회)

### Mac (운영 머신)
```bash
brew install ffmpeg
pip3 install pydub openai-whisper
```

### Windows
```powershell
# 1. Python 3.10+ (python.org 공식 인스톨러 권장 — Store 스텁과 충돌 회피)
# 2. ffmpeg (winget 또는 https://www.gyan.dev/ffmpeg/builds/)
winget install Gyan.FFmpeg
# 3. Python 패키지
pip install pydub openai-whisper
```

> Whisper 첫 실행 시 모델 자동 다운로드(small 기준 ~460MB) — 멈춘 듯 보여도 정상.
> Windows에서 `device: mps` (config.json) 만나면 `step2_whisper_sub.py` 자동 cpu fallback. config 수정 불필요.

---

## 주의사항

- **config.json은 수정 금지** — 운영본은 Mac 기준 (`device: mps`). Windows는 fallback으로 처리됨.
- **NAS 직접 실행** — 로컬 복사하지 않음. 두 OS 모두 `BASE_DIR` 상대경로로 동일 동작.
- **한글 경로**: Mac은 NFD, Windows는 NFC. `m4_run.py`/`utils.py`가 `unicodedata.normalize("NFC", ...)`로 정규화.
- **Premiere 임포트**: `.xml` 파일을 File → Import (저장된 `.prproj` 열기 아님).
- **TTS 미감지 폴더는 자동 스킵** — 마커만 있고 TTS 없으면 처리 목록 제외.
- **여러 프로젝트 한 브랜드**: `_1`, `_2` suffix로 구분 (예: `20260504_자보티바_1`, `20260504_자보티바_2`).
- **동일 브랜드 다중 매칭** 시 자동 진행하지 말고 사용자에게 확인.
