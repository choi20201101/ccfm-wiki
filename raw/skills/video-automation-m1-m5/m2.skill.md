# M2 영상 기획안 생성 스킬

## 설정

```
M1_OUTPUT  = //192.168.0.103/01_마케팅사업부_직원/{NAS_USER}/3팀 영상 자동화/M1/output
M1_DONE    = //192.168.0.103/01_마케팅사업부_직원/{NAS_USER}/3팀 영상 자동화/M1/output/done
LOCAL_WORK = C:/Users/{WINDOWS_USER}/Desktop/클로드
M2_OUTPUT  = //192.168.0.103/01_마케팅사업부_직원/{NAS_USER}/3팀 영상 자동화/M2/output
```

---

## Step 0. 규칙 로드

실행 전 반드시 두 파일 Read:
- `references/plan_rules.md` → 기획안 작성 규칙 내재화
- `references/prompt_rules.md` → 씬 프롬프트 작성 규칙 내재화

---

## Step 1. M1 아웃풋 스캔

```bash
ls "{M1_OUTPUT}/"
```

브랜드명이 인자로 주어지면 해당 폴더만, 없으면 목록 보여주고 선택 요청 (done 폴더 제외).

---

## Step 2. 로컬 복사

```bash
cp -r "{M1_OUTPUT}/{날짜_브랜드}" "{LOCAL_WORK}/{날짜_브랜드}"
ls "{LOCAL_WORK}/{날짜_브랜드}/"
```

---

## Step 3. 인풋 파일 읽기

**브랜드·제품 추출:**  
폴더명 `{날짜_브랜드명}` 또는 `{날짜_브랜드명_제품명}` 에서 파싱.
- `_`로 분리 후 날짜(8자리 숫자) 제외 → 나머지가 브랜드+제품
- 예: `20260511_나잇사렌_세럼` → 브랜드=`나잇사렌`, 제품=`세럼`
- 예: `20260511_나잇사렌` → 브랜드=`나잇사렌`, 제품=`null`

로컬 폴더에서 3개 파일 Read:

```
01_*_brand_asset.md   ← 브랜드 DB
02_*_영상분석.md        ← 크리에이티브 레퍼런스 (아이디어 추출용)
03_user_input.md      ← 사용자 방향성 + 컨셉이미지
```

**멀티 제품 브랜드 처리:**  
`01_brand_asset.md`에 제품이 2개 이상인 경우, `제품` 변수에 해당하는 섹션만 포커스해서 읽는다.  
제품 섹션 이외의 공통 브랜드 정보(브랜드 톤, 타겟, USP 등)는 모두 포함.

02번 파일은 구조·톤·크리에이티브 아이디어 추출용으로만 활용.
M3로 전달하지 않음.

---

## Step 4. CONCEPT_TYPE 판별

`03_user_input.md`의 컨셉이미지 항목 확인.

| 조건 | CONCEPT_TYPE | 기획안 방향 |
|------|-------------|-----------|
| 컨셉이미지 있음 + 비실사 (캐릭터·CGI·일러스트·오브젝트) | `"컨셉"` | 컨셉이미지 스타일·세계관 기준 |
| 컨셉이미지 있음 + 실사 인물 사진 | `"실사"` | 실사 인물 기준 |
| 컨셉이미지 없음 | `"실사"` | 모델이미지 기준 |

CONCEPT_TYPE을 이후 모든 단계에서 유지.

---

## Step 5. 기획안 생성

`references/plan_rules.md` 규칙 적용해 기획안 초안 작성.

**출력 항목 (고정, 순서 변경 금지):**

```markdown
# 영상 기획안 — [브랜드명]
생성일: YYYY-MM-DD

## 컨셉 한 줄
## 전체 영상 길이
## 화자 + 타겟
## 광고 유형
## 포맷
## 스크립트 전문

| # | 구간 | 시간 | 스크립트 | 비고 |
```

스크립트 전문: 모든 씬 행으로 나열, 시간 `MM:SS~MM:SS`, 비고는 특이 연출만.

---

## Step 6. 품질 체크 (3순회)

1순회: 품질 기준 6개 + 7초 규칙 대입 → Fail 목록화
2순회: Fail 씬 재작성
3순회: 전체 흐름 + CTA 강도 최종 확인

---

## Step 7. scenes.json 생성

스크립트 전문 파싱 + `references/prompt_rules.md` 적용.

```json
[
  {
    "id": 1,
    "scene_id": "s01",
    "role": "Hook",
    "start_sec": 0.0,
    "end_sec": 2.0,
    "speech_start_sec": 0.0,
    "script": "스크립트 텍스트",
    "prompt": "씬 프롬프트",
    "refs": ["concept"],
    "sfx": ["SFX_027", "bgm_start"]
  }
]
```

| 필드 | 규칙 |
|------|------|
| `scene_id` | `s01`, `s02` … 영문+숫자만, 한글 금지 |
| `start_sec` / `end_sec` | `MM:SS` → float 초 변환 |
| `speech_start_sec` | 기본값 = `start_sec`. 스크립트 없으면 `null` |
| `script` | 없으면 `null` |
| `prompt` | prompt_rules.md 규칙 적용 |
| `refs` | `"concept"` `"model"` `"product"` 조합 또는 `[]` |
| `sfx` | sfx_library.md 참조 |

---

## Step 8. 로컬 저장

Write 툴로 기획안/scenes.json 저장 직후:

```bash
cp "{LOCAL_WORK}/{날짜_브랜드}"/m1_img_* \
   "{LOCAL_WORK}/{날짜_브랜드}/" 2>/dev/null || true
```

컨셉이미지 없으면 에러 무시.

---

## Step 9. NAS M2/output 업로드

```bash
mkdir -p "{M2_OUTPUT}/{날짜_브랜드}"
cp "{LOCAL_WORK}/{날짜_브랜드}/04_*_기획안.md" "{M2_OUTPUT}/{날짜_브랜드}/"
cp "{LOCAL_WORK}/{날짜_브랜드}/scenes.json" "{M2_OUTPUT}/{날짜_브랜드}/"
cp "{LOCAL_WORK}/{날짜_브랜드}"/m1_img_* \
   "{M2_OUTPUT}/{날짜_브랜드}/" 2>/dev/null || true
ls "{M2_OUTPUT}/{날짜_브랜드}/"
```

확인: `04_기획안.md` + `scenes.json` + 컨셉이미지(있을 경우).

---

## Step 10. M1/output → done 이동

```bash
mkdir -p "{M1_DONE}"
cp -r "{M1_OUTPUT}/{날짜_브랜드}" "{M1_DONE}/"
rm -rf "{M1_OUTPUT}/{날짜_브랜드}"
```
