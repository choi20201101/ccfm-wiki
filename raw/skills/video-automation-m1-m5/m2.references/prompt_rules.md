# 씬 프롬프트 작성 규칙

---

## 화면 다양성 원칙 (필수)

**스크립트(음성)와 화면(비주얼)은 독립적이다.**
말하는 내용이 있다고 해서 화면에 반드시 인물이 카메라를 향해 말하는 장면일 필요 없음.
음성이 설명하는 동안 화면은 그 내용을 시각적으로 가장 효과적으로 보여주는 방식을 선택한다.

| 화면 유형 | 적합한 상황 |
|---------|-----------|
| 인물 직접 발화 | 훅·감정 공감·CTA 등 인물 표정·리액션이 중요한 씬 |
| 제품 사용 장면 | "발랐더니", "써봤더니" 등 사용 경험 언급 씬 |
| 제품 클로즈업 | 용량·성분·패키지 강조 씬 |
| 결과 비주얼 | 효과·변화를 보여주는 씬 |
| 상황/행동 묘사 | 공감 유발 상황 (두피 긁기, 머리 감기 등) |
| 오브젝트·소품 | 약국 진열대, 성분 오브젝트 등 |
| 배경소스(B-roll) | 전환·분위기·긴박감 등 |

> **한 영상 안에서 최소 3가지 이상의 화면 유형이 섞이도록 의식적으로 구성할 것.**
> 모든 씬이 "인물이 카메라에 대고 말하는 장면"이 되어서는 안 됨.

## 컨셉이미지 의도 반영 원칙

scenes.json 작성 전 컨셉이미지를 보고 **앵글·구도·무드**를 파악한다.
기획안 씬 목록을 보면서 컨셉이미지의 앵글이 자연스럽게 맞는 씬에 반영한다.

- 강제 전체 적용 금지 — 기획안 의도를 해치면 쓰지 않는다
- 리뷰어가 직접 말하는 씬 → 카메라 응시 유지 가능
- 제품 사용·결과·신체 비주얼 씬 → 컨셉이미지 앵글 활용 검토

예: 컨셉이미지가 "건강한 모발 뒷모습" → 두피 적용·모발 결과 씬에서 뒷모습/옆모습 앵글 활용.
예: 컨셉이미지가 "클레이 캐릭터 측면" → 제품 등장 씬에서 측면 구도 활용.

## CONCEPT_TYPE별 STYLE_ANCHOR

모든 씬 프롬프트 앞에 붙이는 고정 문구.
ref가 있는 씬에만 붙임. ref 없는 씬은 STYLE_ANCHOR 없이 프롬프트만.

| CONCEPT_TYPE | STYLE_ANCHOR |
|-------------|-------------|
| `"실사"` | `이 사람이 [국가] [장소] [실내/실외]에서 찍은 스마트폰 촬영 이미지처럼,` |
| `"컨셉"` | 컨셉이미지 보고 스타일 한 단어 추출 → `이 {스타일} 컨셉만 유지한채로,` |

### 실사 STYLE_ANCHOR 작성 규칙

**국가** → `03_user_input.md` 또는 `01_brand_asset.md` 톤앤매너 참고. 명시 없으면 기본값 `한국`.

**장소 + 실내/실외** → 씬 prompt 맥락 보고 판단:

| 씬 맥락 | 장소 | 실내/실외 |
|--------|------|---------|
| 거실·집 | 거실 | 실내 |
| 욕실 | 욕실 | 실내 |
| 카페 | 카페 | 실내 |
| 헬스장·필라테스 | 헬스장 | 실내 |
| 사무실 | 사무실 | 실내 |
| 공원·산책로 | 공원 | 실외 |
| 해변·야외 | 야외 | 실외 |
| 맥락 불명확 | 실내 | 실내 |

예시:
```
이 사람이 한국 거실 실내에서 찍은 스마트폰 촬영 이미지처럼,
이 사람이 한국 카페 실내에서 찍은 스마트폰 촬영 이미지처럼,
이 사람이 한국 공원 실외에서 찍은 스마트폰 촬영 이미지처럼,
이 사람이 브라질 해변 실외에서 찍은 스마트폰 촬영 이미지처럼,
```

컨셉 스타일 추출 예시:
- 클레이 피규어 수달 → `클레이 피규어`
- 햄스터 캐릭터 → `햄스터 캐릭터`
- 종이 질감 → `종이 스타일`
- 실사 인물 사진 → CONCEPT_TYPE = "실사"로 분류 (컨셉 아님)

## ref 결정 로직

| 씬 내용 | refs |
|--------|------|
| 모델·인물 등장 (CONCEPT_TYPE = "실사") | `["model"]` |
| 모델·인물 등장 (CONCEPT_TYPE = "컨셉") | `["concept"]` |
| 제품 직접 등장 | `["product"]` 추가 |
| 모델 + 제품 동시 | `["model", "product"]` 또는 `["concept", "product"]` |
| 순수 UI·카운트다운·추상 전환 | `[]` |

> 프롬프트에 제품이 직접 등장하면 씬 유형 무관하게 `["product"]` 포함.

## 고정 규칙 (반드시 지킬 것)

**0. 프롬프트에 텍스트 묘사 절대 금지**
prompt 필드에는 인물·동작·배경·감정 등 시각 장면만 묘사. 화면에 보이는 글자·문구는 편집 단계에서 삽입되므로 prompt에 절대 넣지 않음.

금지 (화면 오버레이): 자막, 자막바, 배너, 타이틀, 숫자 표시, 카운트다운 숫자, UI 텍스트, 가격 문구, CTA 문구 등.
금지 (씬 내 소품): 들고 있는 종이·서류의 내용, 스마트폰·태블릿 화면 속 텍스트, 임상 수치·그래프 수치, 화이트보드·간판·포스터 글씨 등.
→ 소품 자체는 등장 가능. 단, 그 안의 문구·수치는 묘사하지 않음.

허용 (제품 자체): 제품 패키지·라벨·용기에 인쇄된 텍스트는 ref 이미지로 전달되므로 prompt에 언급할 필요 없고, 막아서도 안 됨.

예: ❌ `"99.7% 개선 수치가 적힌 그래프를 가리키며"` → ✅ `"그래프 화면을 가리키며"`
예: ❌ `"스마트폰에 리뷰 문구가 보이며"` → ✅ `"스마트폰 화면을 보며"`

**1. 스크립트 있는 씬 → 그 상황·감정이 화면에 보여야 함**
스크립트가 선언·상황이면 그 장면이 화면에 구현되어야 함.
- "속보" → 뉴스데스크, 앵커 포즈, 뉴스룸
- "비밀 특가" → 카메라에 속삭이며 제품 가리키기
- "바르자마자 따끔" → 실제로 바르는 동작 + 따끔 반응

**1-1. CONCEPT_TYPE = "실사" → 프롬프트에 "말하는" 표현 금지**
실사 이미지 기반 AI 생성 영상은 말하는 장면 퀄리티가 낮음.
- 금지: `카메라를 향해 말하는`, `말하듯이`, `말하며`, `이야기하는`, `설명하는` 등 발화·구술 묘사
- 대체: `카메라를 바라보는`, `카메라를 향해 포즈를 취하는`, `미소 짓는`, `고개를 끄덕이는`, `표정 변화` 등 비언어 표현으로 대체
> CONCEPT_TYPE = "컨셉"이면 이 규칙 적용 안 함 — 컨셉/애니메이션 스타일은 말하는 장면 사용 가능.

**2. CONCEPT_TYPE = "컨셉" → 실물 인물 묘사 금지**
컨셉이미지가 있으면 씬에 등장하는 인물도 컨셉 스타일 캐릭터여야 함. 실사 인물(나이·성별·국적·피부톤 등 현실 인물 묘사) 절대 사용 금지.
- ❌ `"30대 한국 여성이 제품을 들고"` → ✅ `"클레이 피규어 캐릭터가 제품을 들고"`
- ❌ `"모델이 카메라를 바라보며"` → ✅ `"햄스터 캐릭터가 카메라를 바라보며"`
> CONCEPT_TYPE = "실사"일 때는 이 규칙 적용 안 함.

**2-1. CONCEPT_TYPE = "실사", ref 없는 씬에 신체 등장 → 인종·나이·성별 명시**
ref가 없으면 AI가 인물을 임의 생성.
- `30대 한국 여성`, `20대 한국 여성` 수준으로 명시.

**3. 제품 등장 씬 → 동작까지 명시**
제품이 화면에 나오면 어떻게 나오는지 써야 함.
- 손에 들고 / 선반에 놓여 / 바르는 중 / 여러 통 쌓인 / 카메라에 내밀며

**4. 장소·배경 항상 명시**
어디서 찍히는 씬인지 없으면 AI가 임의로 만듦.
- 야외 자연광 / 욕실 거울 앞 / 집 전신 거울 앞 / 식탁 위 / 뉴스룸 / 카페 등

**5. 연속된 행동은 병합**
1~2초 내 같은 행동 흐름이면 씬 나누지 말고 하나로.

## 배경소스 씬 원칙

배경소스 씬(편집자가 UI를 올릴 배경)은 스크립트와 1:1 매칭 불필요.
씬의 감정·상황이 전달되는 장면이면 됨.
장소·상황은 반드시 명시.

아래는 참고용 예시 (강제 아님):
- 가격·할인 긴박감 → 지갑·카드 꺼내는 손 클로즈업
- 카운트다운·시간 압박 → 스마트폰 빠르게 터치하는 손
- 성분·효능 강조 → 제품 + 성분 오브젝트 클로즈업 슬로우모션
- 공감·감정 → 모델 자연스러운 일상 행동
- 사회증거·재구매 → 제품 여러 개 손으로 집어드는 장면
- 순수 전환·분위기 → 씬 색감에 맞는 추상 빛·질감 클로즈업
