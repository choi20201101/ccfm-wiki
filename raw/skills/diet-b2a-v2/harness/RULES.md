# Harness RULES v2

## R1. 멱등
- 각 세트·스텝 완료 시 `state.json`에 플래그. 재실행 시 완료된 부분 skip
- Gemini/Kling 결과 파일 존재 + 사이즈 > 최소값 → 재생성 금지

## R2. Gemini 세션 보호
- `.session/` 삭제 금지 (로그인 재수행 비용)
- 실행 중 SingletonLock 파일 생성 → 강제 종료 시 수동 삭제 필요

## R3. 인코딩
- 모든 텍스트 UTF-8, Windows cp949 금지
- 프롬프트에 em-dash(—), 스마트 따옴표, 이모지 금지

## R4. 프롬프트 잠금
- `steps/01-styles-prompts/prompts/setN/*.txt` 파일 단위 교체만 허용
- 런타임 f-string 치환 금지

## R5. 병렬 제출 제한
- Kling concurrent task 상한 미확인 → 순차 2초 간격 제출
- 서브미션 실패 시 30/60/120s 백오프

## R6. 레이아웃 하드 한계
- 캔버스 1080×1920
- 체중계 박스는 화면 밖 금지
- 자막·박스 픽셀 중첩 금지

## R7. 컷 지점 BGM 정렬
- 영상2: 4.85s ± 0.3s (DW3 드롭 기준)
- 영상3: 4.0s (드롭 2.5s는 후반이 너무 짧아 타협)

## R8. 얼굴 모자이크 강제
- 모든 Kling 원본 → 모자이크 필터 통과
- 박스 112×125 내외, 얼굴만 덮기

## R9. 컨텐츠 안전
- AI 생성 인물만 사용
- 약/시술 언급 금지
- 음원은 레퍼런스 mp3 사용 (업로드 시 플랫폼 공식 음원 교체 권장)

## R10. eval 피드백 루프
- 세트별 썸네일 1장씩 자동 추출 → `eval_preview/*.jpg`
- 품질 미달 세트는 해당 스텝만 재실행
