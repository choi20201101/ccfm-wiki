잠재고객 (리타겟): 유입/행동 기반 직접 생성 세그먼트
장점:
구매 전환 가능성이 높은 사용자에게 집중 가능
이미 브랜드 또는 제품에 방문/반응한 적 있는 사용자이므로 구매 전환 가능성이 높음
낭비가 적고 예산 효율적
신규 유입 타겟에 비해 불필요한 노출이 적어 비용 대비 효율이 우수
구체적 행동 기반 타겟 설정 가능
장바구니, 회원가입 등 행동 기반 세분화 타겟팅 가능
브랜드 신뢰도 및 친숙도 상승
반복 노출을 통해 사용자와의 심리적 거리감 축소, 브랜드 각인 효과 기대
단점:
신규 고객 유입 감소
리타겟팅은 이미 우리 브랜드/제품을 인지하거나 상호작용한 사용자에게 집중되는데, 여기에 과도한 예산을 투입하면 새로운 잠재 고객을 발굴하고 유입시키는 활동에 투입될 예산이 감소함
이는 장기적으로 신규 고객 유입 풀을 고갈시켜 지속적인 성장을 저해하게 됨
성장 정체
단기적으로는 높은 ROAS를 유지하는 것처럼 보일 수 있으나, 신규 고객 유입 없이는 전체적인 매출 규모나 시장 점유율 확대에 한계 발생
결국 한정된 리타겟팅 모수 내에서의 성과에 갇히게 됨
광고 피로도 심화
제한된 모수에게 광고가 집중 노출되면서 광고 피로도가 빠르게 증가하고, 이는 클릭률(CTR) 하락 및 광고 효율 저하로 이어짐
자동타겟 확장: 머신러닝 기반 유사 오디언스 자동 확장
너무 넓은 타겟은 효율을 망칠 수 있습니다
예산이 일 30만 원 수준인데 타겟 모수가 500만이면
cpc가 낮고 확산성이 좋은 지면이더라도
충분한 테스트가 이뤄질 만큼의 결과는 나오기 어렵습니다
무의미한 테스트로 끝나거나 비효율적인 예산 소진으로 이어질 수 있습니다
적정 타겟 범위는 100만 이하로 생각하면 좋습니다
모수를 좁혀서 운영하는 이유는 예산을 낭비하지 않고 빠르게 고효율 타겟을 발굴할 수 있기 때문입니다
특히 예산이 작고, 아직 성과가 확보되지 않은 초기라면 더 뾰족하게 타겟을 좁혀 운영해야 합니다
성별 구분: 남성과 여성은 반응 요소가 다를 수 있으므로 분리 운영
연령대 구분: 2030 vs 4050은 구매 동기나 관심 키워드가 다름
주력 타겟: 스킨케어, 메이크업 제품 등 타이트하게 구분
서브 타겟: 주력 타겟 + @ (소비자 행태 기반으로 생각하고 가설 잡아 테스트 필요)
지면 구분: 피드 / 메인 / 서브 / 스마트채널 등 성격이 다른 지면은 따로 성과 비교
(성과가 나타난 타겟 또는 지면을 기준으로, 점진적 확장 전략도 필요합니다)
넓은 타겟을 활용할 경우에는 다음을 꼭 주의해야 합니다
모수가 많기 때문에 낮은 입찰가로 노출 가능하나 빠르게 소진될 위험이 있습니다
타이트한 모니터링이 필수이며 빠르게 효율이 안 나오는 그룹은 과감히 OFF 판단 필요합니다
다만 맞춤타겟에 자사몰 키워드만 집중할 경우
리타겟팅 성격의 유저 비중이 높아져서
신규 유입 타겟 확보가 어렵고 확장성 떨어짐
따라서 다양한 유형의 키워드를 섞어 신규 고객 확보 + 확장 테스트를 병행해야 합니다

### 오디언스 세그먼트 설정 방법
오디언스 타겟팅: 사용자의 행태 정보를 기반으로 한 타겟팅
GFA 타겟팅은 커뮤니케이션 애드를 제외하고는 오디언스로 진행합니다
캠페인 생성 → 광고 그룹 생성 → 타게팅 유형 선택
[IMG]
콘텍스트 타겟팅: 노출될 지면의 주제를 파악하여 설정한 주제와 매칭하여 노출하는 타겟팅
소비자 행태 기반이 아닌 주제 기반으로 노출되는 커뮤니케이션 지면만 활용 가능
캠페인 생성 → 광고 그룹 생성 → 타게팅 유형 선택
[IMG]
상세 타겟 - 구매의도: 특정제품/서비스를 구매할 의도를 가진 것으로 추정된 사용자 타겟팅
GFA에서 가장 많이 활용하는 메인 타겟
캠페인 생성 → 광고 그룹 생성 → 관심사/구매의도 설정하기
[IMG]
상세 타겟 - 관심사: 유저의 관심 분야, 흥미, 성향 정보를 타겟팅
구매의도 다음으로 많이 활용하는 타겟
캠페인 생성 → 광고 그룹 생성 → 관심사/구매의도 설정하기
[IMG]
자동타겟 확장: 설정된 타겟 이용자 뿐 아니라 머신러닝을 통해 광고에 보다 적합한 이용자 발굴
타겟 확장이 적용되는 지면은 네이버 피드 영역 게재 위치의 일부이며 운영 중 변경 가능합니다
리포트에서 따로 구분하여 성과 체크는 불가합니다
캠페인 생성 → 광고 그룹 생성 → 타겟 확장 설정
[IMG]
확장데모: 영유아~초등학생 고학년 부모까지 설정 가능한 타겟
자녀가 있는 부모 대상으로 타겟팅이 가능합니다
다른 타겟과 동시 테스트 진행 가능합니다
캠페인 생성 → 광고 그룹 생성 → 확장 데모 설정
[IMG]
지역타겟 설정: “~시 ~구 ~동”까지 타겟팅 가능함
전국으로 배송되기에 많이 활용하지는 않는 타겟
다만 “집중력 향상 제품을 판매한다면” 대치동 같은 비싼 동네를 가설 잡아 테스트 진행 가능
캠페인 생성 → 광고 그룹 생성 → 지역 타겟 설정하기
[IMG]
타겟 설정은 무조건 업종별 타겟만 설정하는 것이 아닌 다양한 테스트가 필요합니다.
머신러닝 기반으로 운영되는 메타/구글은 매체 자체적으로 광고에 반응할 만한 소비자를 학습하고 찾아가지만 GFA는 머신러닝 보다는 타겟팅을 주력으로 운영하는 매체입니다
예를 들어 식품 광고를 한다고 말했을 때 관심사: 식품 / 구매의도: 식품 타겟만
 진행하는 것이 아닌 식품 안에서도 음료, 가공식품 등으로 구분하여 테스트 진행도 가능하며 해당 식품에 관심을 가질법한 타겟을 설정하여 테스트 진행이 가능합니다.
예시 제품: 바디워시
주력 타겟: 바디케어
서브 타겟: 헤어케어, 뷰티, 향수, 주거관련
예시 제품: 프라이팬
주력 타겟: 가정/생활, 주거관리용품
서브 타겟: 이사, 식품, 결혼
