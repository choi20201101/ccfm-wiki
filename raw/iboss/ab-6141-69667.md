# AI 개발 전 보면 100시간 아끼는 바이블 (치트키) 꼭 보세요

## 핵심 인사이트
- "설계가 90%, 코드는 출력물이다" - 마크다운 문서 설계에 30분 투자하면 3시간 삽질을 방지
- MECE 원칙 적용: 각 Step이 자기 영역만 담당하면 문제 발생 시 해당 부분만 확인 가능
- 입출력 파일명 통일의 중요성 - "파일명이 곧 Step 간 계약", 형식 미정의 시 연쇄 오류 발생
- 3개 이상 연쇄 수정 필요 시 코드 부분 수정이 아닌 전면 재시작이 10배 빠름
- SPEC.md는 100줄 이내 유지 - "AI는 앞뒤를 우선 처리하고 중간이 약해지는 특성"
- 생성형 AI 활용 시 모델 명시 필수 - 모델 미지정 시 저품질 오픈소스 자동 선택
- 비개발자도 한국어로 마크다운 수정 후 AI에 재생성 요청하면 유지보수 가능
- 멀티 에이전트 병렬 개발 - 순차 개발 대비 4배 빠른 개발 속도 달성
